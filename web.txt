NAMA PROYEK: Global Supply Chain Risk Intelligence Platform[cite: 1]
DESKRIPSI: Platform berbasis web untuk memantau risiko rantai pasok global (logistik, cuaca, geopolitik, dan ekonomi) guna mendukung keputusan ekspor/impor perusahaan.[cite: 1]

1. ARSITEKTUR & TEKNOLOGI
Aplikasi ini akan dibangun menggunakan arsitektur Monolith (satu *codebase*).
- Backend: PHP dengan framework Laravel.[cite: 1]
- Database: MySQL.[cite: 1]
- Frontend: Blade Templates bawaan Laravel, dikombinasikan dengan Bootstrap 5.[cite: 1]
- Interaksi Data: AJAX dan JavaScript ES6 untuk memanggil data tanpa reload halaman.[cite: 1]
- Visualisasi: Chart.js untuk grafik data dan Leaflet.js untuk peta interaktif.[cite: 1]

2. INTEGRASI API EKSTERNAL (GRATIS)
Sistem harus menarik data dari sumber berikut:
- Open-Meteo API: Data cuaca global (suhu, hujan, angin, risiko badai).[cite: 1]
- World Bank API: Data makroekonomi (GDP, inflasi, populasi, ekspor/impor).[cite: 1]
- REST Countries API: Profil wilayah (nama negara, mata uang, bahasa).[cite: 1]
- ExchangeRate API: Kurs mata uang real-time.[cite: 1]
- World Port Index: Data publik lokasi pelabuhan dunia.[cite: 1]
- GNews API: Berita ekonomi, logistik, dan geopolitik.[cite: 1]
- OpenStreetMap API: Menyediakan data ubin peta dasar (map tiles) yang akan digunakan bersama pustaka Leaflet.js untuk memvisualisasikan peta global secara interaktif.[cite: 1]

3. KEBUTUHAN DATABASE
Aplikasi membutuhkan sekitar 15-20 tabel.[cite: 1] Berikut adalah tabel inti yang wajib dibuat:
- `users`: Data admin/pengguna.[cite: 1]
- `countries`: Profil dasar negara.[cite: 1]
- `risk_scores`: Riwayat skor risiko negara.[cite: 1]
- `news_cache` & `articles`: Penyimpanan sementara data berita dari API untuk menghindari limit request.[cite: 1]
- `ports`: Titik koordinat pelabuhan.[cite: 1]
- `watchlists`: Daftar negara yang difavoritkan/dipantau pengguna.[cite: 1]
- `positive_words` & `negative_words`: Kamus statis untuk analisis sentimen berita.[cite: 1]

4. LOGIKA ANALISIS & ALGORITMA KHUSUS (DIBUAT MANUAL, TANPA AI)
- Analisis Sentimen (Lexicon-Based): Buat fungsi PHP yang memecah teks berita dari API, lalu mencocokkan kata-katanya dengan tabel `positive_words` dan `negative_words` untuk menentukan apakah berita bersentimen positif atau negatif.[cite: 1]
- Algoritma Prediksi Risiko: Buat sistem bobot (Weighted Risk Model) yang menjumlahkan persentase risiko cuaca, inflasi, mata uang, dan sentimen berita menjadi satu skor tunggal (Misal: Total Skor = 28% / Low Risk).[cite: 1]

5. PEMBUATAN API INTERNAL (REST)
Developer wajib membuat *endpoint* API internal berikut yang akan dikonsumsi oleh AJAX di bagian Frontend:[cite: 1]
- GET /api/countries[cite: 1]
- GET /api/risk[cite: 1]
- GET /api/ports[cite: 1]
- GET /api/news[cite: 1]
- GET /api/currency[cite: 1]

6. DAFTAR FITUR DASBOR UTAMA
- Dasbor Negara & Pembanding: Menampilkan profil (cuaca, GDP, inflasi) suatu negara dan fitur untuk membandingkan dua negara secara berdampingan.[cite: 1]
- Peta Cuaca & Pelabuhan Interaktif: Menampilkan penanda/marker pelabuhan dan kondisi badai/hujan di atas peta dunia menggunakan Leaflet.js dan OpenStreetMap.[cite: 1]
- Intelijen Berita & Mata Uang: Menampilkan sentimen berita terkini dan grafik tren perubahan nilai tukar.[cite: 1]
- Dasbor Admin: Mengelola data pengguna, set data pelabuhan, dan artikel.[cite: 1]
